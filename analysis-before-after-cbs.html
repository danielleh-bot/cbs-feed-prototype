Key Advantages

Why Crawler-Generated Brand Kits Win

Moving from manual custom properties to automated brand kit extraction provides compounding benefits across accuracy, scale, and maintenance.

🎯

Pixel-Perfect Accuracy

The crawler extracts actual computed CSS values from the live DOM — not approximations from eyedropping screenshots. CBS's #181818 charcoal is captured exactly, not rounded to #333333. Font stacks include the real Publico Headline with fallback chain.

🔀

Semantic Richness

Custom properties are flat key-value pairs. A brand kit JSON captures meaning: CBS uses serif for editorial content and sans-serif for UI. Metadata is always uppercase with tracking. These rules let the feed render semantically-aware cards no flat property list can express.

⚡

Setup in Minutes, Not Hours

A full brand kit for CBS News — covering 3 font families, 7 core colors, 7-level type hierarchy, image treatments, and component patterns — was generated in under 5 minutes. Manual configuration takes 4–6 hours and still misses semantic tokens.

🔄

Auto-Sync on Redesign

When CBS updates their fonts, shifts a color, or changes their card style, a re-crawl captures it immediately. Manual properties go stale silently — the feed drifts from CBS's design with no alert, no diff, no audit trail.

📈

Scales to 3,500+ Publishers

Taboola serves 3,500+ publishers. Manually configuring custom properties for each is an enormous ongoing cost. Automated crawling can generate and maintain brand kits for the entire publisher network in a single scheduled pipeline run.

🛡

Captures What Properties Can't

Manual custom properties can set fontFamily and titleColor. They cannot express: "headlines use a display serif while UI uses sans-serif," "all metadata is uppercase with 0.12em tracking," or "every image uses sharp 0px corners." Brand kit JSON can.

Custom UI Modes

Replacing Custom UI Modes at Scale

Taboola currently manages hundreds of custom UI modes — per-publisher visual override configurations that account managers build and maintain in Backstage. These modes exist because the default feed styling doesn't match publisher branding. The brand kit approach automates what UI modes do manually, eliminating ~70% of them.

What Custom UI Modes Do Today

Each UI mode is a bespoke configuration bundle that overrides Taboola's default feed appearance for a specific publisher. Account managers create these by visually inspecting publisher sites, extracting design values by hand, and encoding them into mode definitions. Common overrides include:

Font Overrides
font-family, font-size, font-weight, line-height for headlines, descriptions, labels
Color Theming
text colors, accent colors, background colors, badge colors, hover states
Card Layout
grid gaps, padding, margins, card density, vertical vs horizontal layout
Thumbnail Styling
border-radius, aspect-ratio, object-fit, hover zoom effects
Label Treatments
text-transform, letter-spacing, badge styles, "Sponsored" vs "Paid"
Interactive States
hover colors, transitions, active indicators, focus rings
